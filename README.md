# 📌 Abstraindo um Bootcamp com OO – Versão Evoluída

Este projeto foi desenvolvido a partir de um desafio da DIO e posteriormente refatorado e evoluído com foco em organização arquitetural e boas práticas de desenvolvimento backend.

### 🔧 Melhorias Implementadas
 - ✔ Separação de responsabilidades em camadas:**
   - Repository – Persistência simulada utilizando Map
   - Service – Regras de negócio
   - View – Interação com usuário via terminal

-  Simulação de persistência de dados (in-memory database)**
-  Implementação de regras de negócio desacopladas da camada de apresentação**
-  Interação com usuário via prompt de comando
-  Estrutura preparada para futura integração com banco de dados real

### 🧠 Conceitos Aplicados
- Programação Orientada a Objetos
- Princípio da Responsabilidade Única (SRP)
- Separação de camadas (Layered Architecture)
- Encapsulamento e abstração
- Simulação de persistência

### 🎯 Objetivo da Evolução
Transformar um projeto inicialmente didático em uma estrutura mais próxima de uma aplicação backend real, aplicando organização arquitetural e separação clara entre domínio, regras de negócio e persistência.
